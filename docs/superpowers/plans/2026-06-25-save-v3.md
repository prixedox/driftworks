## Task 3: Update main.ts — SAVE_KEY, migration, readSave, writeSave

**Files:** Modify `src/main.ts`.

**Interfaces consumed:** `SaveState` (v3), `Snapshot` (now has `upgrades`).

**Design decision on migration:** v2 saves live under the key `'driftworks.save.v2'`. v3 saves will live under `'driftworks.save.v3'`. On first load, the app will look for a v3 save first; if absent, it checks for the old v2 key and migrates it in memory to v3 (applying defaults for the new fields), then writes it under the new key. The old v2 key is left in localStorage (not deleted) to avoid any data loss on edge cases.

### Current relevant code in `main.ts` (verbatim):

```ts
const SAVE_KEY = 'driftworks.save.v2';

function readSave(): SaveState | null {
  try {
    const raw = localStorage.getItem(SAVE_KEY);
    if (!raw) return null;
    const s = JSON.parse(raw) as SaveState;
    return s && s.version === 2 && Array.isArray(s.modules) ? s : null;
  } catch {
    return null;
  }
}
```

```ts

{
  function migrateToV3(raw: unknown): SaveState | null {
    if (!raw || typeof raw !== 'object') return null;
```
